# CLAUDE.md — working in this repo

Guidance for Claude Code (and other AI tools) working in the **florina-elixir** repo.
For general Elixir/Phoenix conventions, see `AGENTS.md` (generated by Phoenix).

## What this project is

The **Elixir / Phoenix / Oban port** of the Florina sales-assistant app — a separate,
isolated codebase from the live Django app (`neurony/florina`). It grows via a
**strangler-fig** strategy: move the product across one slice at a time, never a big-bang
rewrite. The first real slice will be the **realtime features** (voice calls + live agent chat).

The current architecture reference is `docs/APPLICATION-SPEC.md` — read it before substantial work.

## Safety

This repo is a sandbox and does not auto-deploy. Even so, before any destructive action,
confirm the git remote is the sandbox fork (`alexandruradulescu-neurony/florina-elixir`),
**never** the live `neurony/florina`.

## How the maintainer works

- **Explain in plain English first.** The maintainer builds with Claude Code but does not read
  code fluently — lead with a no-jargon explanation before any code or diff.
- **Plan before doing**, and get approval before anything destructive or wide-reaching.
- **Work on `develop`; never push or merge to `main` unasked** (see Branching & deploy).

## Branching & deploy

- **`main` is production — Railway auto-deploys it. Every change to `main` is a LIVE deploy.**
- Do all work on **`develop`** (or feature branches off `develop`). Pushing `develop` is safe — it does **not** deploy.
- Merge/deploy `develop` → `main` **only on the maintainer's explicit green light.** Treat it as shipping to live.

## Commands

```bash
mix setup          # deps + create/migrate DB + build assets
mix phx.server     # run at http://localhost:4000
mix precommit      # compile (warnings as errors) + format + unused-deps check + tests
```

Local Postgres connects as the macOS user with no password (see `config/dev.exs`).

## Conventions

- **Tests are local-only and never committed** — the `test/` folder is git-ignored. CI runs
  compile + format + unused-deps checks, not the test suite.
- Background jobs use **Oban** (see `lib/florina/workers/`).
- Secrets live in `.env` (git-ignored); the voice API keys are already there.
